What is the multi-run harvesting?

The multi-run harvesting aims to combine statistics from multiple runs into one single output and use them for Data Qualilty Monitoring as well as producing alignment and calibration results based on broader input. Documentation on how to technically perform the DQM multi-run harvesting can be found here: https://twiki.cern.ch/twiki/bin/viewauth/CMS/DQMOffline#MultiRun

How does it work?

The service displays the results of the performed processing for the Prompt Calibration Loop. Provided multi-run infrastructure for PCL resides on top of the Tier0 implementation and CMSSW, being an independent work.

Multi-run states

The multi-run can be assigned one of a few states. The state machine and possible transitions are shown in the picture.

To improve the readability the the state can be identified by the color of the entry, what can be consulted with the legend. There are three end states:

  • Processing failed - AlCa Harvesting fails continuously, manual supervision required
  • DQM Upload OK - success, no payload upload is forseen
  • Uploads OK - full success

Retries

The AlCa Harvesting step and the Uploads can be repeated after failure occurs. This will happen automatically until the maximum failures threshold is exceeded. The threshold can be configured for the specific deployment.
